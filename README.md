
# OpenWrt Psiphon-Core Setup & Automation Guide

این پروژه یک راهنمای جامع و اسکریپت هوشمند برای راه‌اندازی، مدیریت و جفت‌کردن هسته لینوکسی سایفون (`psiphon-core`) با روترهای مبتنی بر **OpenWrt** و **GL.iNet** (به ویژه پردازنده‌های ۶۴ بیتی با معماری `aarch64_cortex-a53`) است.

به دلیل محدودیت‌های دسترسی، سایفون در هر بار اجرا پورت‌های SOCKS و HTTP کاملاً تصادفی (Random) باز می‌کند. این اسکریپت به صورت داینامیک پورت‌ها را از لاگ سیستم شکار کرده و آن‌ها را روی پورت‌های ثابت شبکه داخلی لایو می‌کند.

---

## 🛠️ ۱. آموزش کامپایل فایل باینری (روی کامپیوتر)

اگر می‌خواهید خودتان فایل اجرایی سایفون را برای معماری روتر (`arm64`) بسازید، ترمینال کامپیوتر را باز کرده و این دستورات را اجرا کنید:

```bash
# ۱. دریافت سورس کد رسمی هسته سایفون از مخزن Psiphon-Labs
git clone [https://github.com/Psiphon-Labs/psiphon-tunnel-core.git](https://github.com/Psiphon-Labs/psiphon-tunnel-core.git)

# ۲. ورود به پوشه اصلی پروژه (کدها مستقیماً در ریشه این مخزن هستند)
cd psiphon-tunnel-core

# ۳. دانلود وابستگی‌ها و تنظیم متغیرهای محیطی برای لینوکس ۶۴ بیتی ARM
go mod download
export GOOS=linux
export GOARCH=arm64

# ۴. کامپایل فوق‌العاده بهینه و سبک برای روتر
go build -ldflags="-s -w" -o psiphon-core .

```

فایل ساخته شده با نام `psiphon-core` را از طریق SFTP (برنامه‌هایی مثل MobaXterm یا WinSCP) به مسیر `/usr/bin/` روتر منتقل کنید.

## 🚀 ۲. آماده‌سازی و نصب پیش‌نیازها (روی روتر)

این دستورات را در ترمینال روتر اجرا کنید تا دسترسی‌ها صادر شده و ابزار پیش‌نیاز `socat` نصب شود:

```bash
# مجاز کردن فایل باینری و ساخت پوشه دیتا
chmod +x /usr/bin/psiphon-core
mkdir -p /usr/bin/psiphon_data

# نصب ابزار socat جهت ریدایرکت ترافیک
opkg update && opkg install socat

```

## 📝 ۳. ساخت فایل کانفیگ عمومی سایفون (`psiphon.config`)

دستور زیر را به صورت کامل کپی کرده و در ترمینال روتر پیست کنید تا فایل تنظیمات استاندارد با پارامترهای عمومی و بدون اطلاعات شخصی در مسیر مشخص شده ساخته شود:

```bash
cat << 'INPUT_EOF' > /usr/bin/psiphon.config
{
  "SocksProxyPort": 10808,
  "HttpProxyPort": 10809,
  "ClientPlatform": "Windows_10.0.26200_11",
  "ClientVersion": "187",
  "DataRootDirectory": "/usr/bin/psiphon_data",
  "EgressRegion": "US",
  "PropagationChannelId": "0000000000000000",
  "SponsorId": "0000000000000000",
  "ServerEntrySignaturePublicKey": "sHuUVTWaRyh5pZwy4UguSgkwmBe0EHtJJkoF5WrxmvA=",
  "UseIndistinguishableTLS": true
}

INPUT_EOF

```

## ⚡ ۴. اسکریپت راه‌انداز داینامیک و هوشمند (روی روتر)

> ⚠️ **نکته بسیار مهم:** اگر آی‌پي داخلی روتر شما چیزی غیر از `192.168.0.1` است، در خطوط مربوط به `socat` (بخش ۵)، آی‌پي روتر خودتان را جایگزین کنید.

کد زیر را کپی کرده و در ترمینال روتر اجرا کنید تا سایفون استارت خورده و پورت‌های شبکه فعال شوند:

```bash
# ۱. بستن پروسه‌های قدیمی برای جلوگیری از تداخل پورت‌ها
killall -9 psiphon-core socat 2>/dev/null

# ۲. اجرای هسته اصلی سایفون در پس‌زمینه
/usr/bin/psiphon-core -config /usr/bin/psiphon.config -dataRootDirectory /usr/bin/psiphon_data > /tmp/psiphon.log 2>&1 &

# ۳. چند ثانیه صبر تا سایفون تانل را برقرار و پورت‌های داخلی را باز کند
echo "در حال راه‌اندازی سایفون... ۷ ثانیه صبر کنید..."
sleep 7

# ۴. استخراج خودکار پورت‌های رندوم سایفون از داخل لاگ فایل
SOCKS_PORT=$(http_port=$(cat /tmp/psiphon.log | grep -o '"port":[0-9]*' | cut -d':' -f2); echo $http_port | awk '{print $1}')
HTTP_PORT=$(http_port=$(cat /tmp/psiphon.log | grep -o '"port":[0-9]*' | cut -d':' -f2); echo $http_port | awk '{print $2}')

echo "پورت‌های شناسایی شده -> SOCKS: $SOCKS_PORT | HTTP: $HTTP_PORT"

# ۵. برقراری پل ارتباطی توسط socat به آی‌پي روتر (پورت‌های ثابت ۱۰۸۰۸ و ۱۰۸۰۹)
socat TCP-LISTEN:10809,fork,bind=192.168.0.1 TCP:127.0.0.1:$HTTP_PORT &
socat TCP-LISTEN:10808,fork,bind=192.168.0.1 TCP:127.0.0.1:$SOCKS_PORT &

# ۶. باز کردن فایروال روتر برای قبول دسترسی از شبکه داخلی
iptables -I INPUT -p tcp -i br-lan --dport 10808:10809 -j ACCEPT

echo "سایفون با موفقیت با پورت‌های ثابت روی شبکه داخلی جفت شد!"

```

## 🛑 ۵. دستورات غیرفعال‌سازی و خاموش کردن کامل سایفون

هر زمان که خواستید سایفون و پل ارتباطی آن را کاملاً متوقف کنید و پورت‌های باز شده را ببندید، این دستور را در ترمینال روتر اجرا کنید:

```bash
# ۱. متوقف کردن هسته سایفون و پروسه‌های socat
killall -9 psiphon-core socat 2>/dev/null

# ۲. پاک‌سازی قوانین فایروال جهت بستن پورت‌های ورودی
iptables -D INPUT -p tcp -i br-lan --dport 10808:10809 -j ACCEPT 2>/dev/null

echo "سایفون و پل‌های ارتباطی با موفقیت غیرفعال و خاموش شدند."

```

## 💻 ۶. تنظیمات کلاینت‌ها (ویندوز، گوشی، مرورگر)

پس از اجرای موفقیت‌آمیز مراحل بالا، برای استفاده از اینترنت بدون فیلتر کافیست مشخصات پروکسی زیر را در افزونه‌های مرورگر (مانند FoxyProxy) یا تنظیمات پروکسی سیستم‌عامل وارد کنید:

* **Proxy Type:** `HTTP`
* **IP Address:** `192.168.0.1` (آی‌پي داخلی روتر شما)
* **Port:** `10809`
EOF

```

برای راه‌اندازی این سناریو روی روتر، به دو فایل اصلی نیاز داری که آدرس و مسیر قرارگیری آن‌ها در سیستم‌عامل روتر (لینوکس OpenWrt) دقیقاً به شرح زیر است:

۱. فایل اجرایی اصلی (Binary File)
این همان فایل برنامهٔ سایفون است که باید با معماری پردازنده روترت (که برای GL-MT3000 از نوع MediaTek Filogic MT7981 با معماری ARM64 یا همان aarch64_cortex-a53) سازگار باشد.

نام فایل: psiphon-core

مسیر قرارگیری در روتر: /usr/bin/psiphon-core

۲. فایل تنظیمات (Configuration File)
این فایلی است که مشخصات کلاینت، آیدی‌ها و پورت‌های ثابت پروکسی را درون آن ذخیره کردیم و اسکریپت در زمان اجرا آن را می‌خواند.

نام فایل: psiphon.config

مسیر قرارگیری در روتر: /usr/bin/psiphon.config

۳. پوشه دیتابیس داخلی (Data Directory)
علاوه بر فایل‌ها، سایفون برای ذخیره اطلاعات سرورها، کش (Cache) و لیست‌های به‌روزرسانی به یک پوشهٔ اختصاصی نیاز دارد که در کانفیگ آدرسش را ست کرده‌ایم:

مسیر پوشه در روتر: /usr/bin/psiphon_data/

۴. فایل لاگ موقت (Log File)
این فایل به صورت خودکار در حافظه موقت روتر ساخته می‌شود تا اسکریپت بتواند پورت‌های رندوم سایفون را از داخل آن پیدا کند. نیازی نیست خودت آن را بسازی:

مسیر فایل لاگ: /tmp/psiphon.log

نکته برای انتقال: با برنامه‌هایی مثل MobaXterm یا WinSCP (از طریق پروتکل SFTP یا SCP) می‌توانی به روتر متصل شوی و فایل اجرایی psiphon-core را مستقیماً در مسیر /usr/bin/ آپلود کنی.
```
